아카교육 → 아카소프트(주)

과학 숏폼 학습 플랫폼 'ProblemShorts' 단독 기획·설계·개발 및 운영 (2024.11 ~ 재직중)

기획부터 아키텍처 설계, 백엔드 개발, 인프라 구성, 운영까지 전 분야를 외부 도움 없이 직접 주도한 첫 프로젝트. 아카교육에서 프리랜서로 기획·구축한 ProblemShorts가 성과를 인정받아 법인(아카소프트) 설립으로 이어지며 합류, 동일 프로젝트를 운영·고도화로 이어감.

  • 아키텍처 설계: DDD(Domain Driven Design), Hexagonal Architecture 기반 RESTful API 구조 설계 및 구축
  • 데이터베이스: PostgreSQL 데이터 모델링 직접 수행
  • 확장 가능한 인프라: 초기 구축 단계부터 동시 접속 1,000명을 목표로 ECS Auto Scaling, RDS Read Replica 기반 수평 확장 구조를 선제 설계. Spring Boot·Next.js 기술 스택 선정 및 CI/CD 자동화까지 초기 환경 직접 구성
  • IaC: AWS CloudFormation 기반으로 인프라를 코드화(생성형 AI를 보조로 활용해 템플릿 작성·반영)
  • 캐싱: Redis로 대시보드 통계, 이메일·본인 인증 등 지연·부하에 민감한 처리를 캐싱
  • 미디어 파이프라인: 숏폼 영상을 S3·CloudFront로 제공하고 AWS MediaConvert로 HLS 등 스트리밍 포맷으로 변환 처리
  • 결제 시스템: 토스페이먼츠 일반결제·정기결제(구독)를 계약 단계부터 직접 연동, 구글·애플 인앱결제까지 구축
  • AI 적재 파이프라인: Gemini API로 PDF 문제를 파싱·전산화하는 적재 파이프라인 구축 — 1,000문제 기준 약 80시간 걸리던 수작업 전산화를 20분 내로 단축(이후 문제당 2분 내외 검수만 사람이 담당). 총 2.3만 문제 중 1.3만 건을 2주 만에 적재(사람 1인 하루 100~160문제 처리 대비 대폭 단축)
  • AI 콘텐츠 생성: Grok으로 문제 해설 숏폼 콘텐츠를 자동 생성(300개 이상 제작), 나노바나나(이미지 생성 AI)로 문제 삽화(figure)를 업스케일·재생성해 시인성 개선
  • 테스트 전략·빌드 최적화: Application Layer(UseCase) 단위 테스트 1,000개 이상 운영. 테스트가 늘며 Spring Context 로딩이 빌드를 지연시키자, Context 로딩 없이 Mockito 기반으로 검증하도록 표준화한 커스텀 어노테이션(LenientMockitoExtension)을 직접 만들어 전환 → 빌드 시간 3분 내외로 유지
  • 운영·모니터링: 상용 주 1회 정기 배포, CloudWatch 지표 수집 + Slack 경보 연동으로 실시간 장애 탐지(장애 발생 시 12시간 내 원인 규명을 목표로 운영)
  • 운영 판단·리뉴얼: 초기 B2C+B2B 모델을 운영·마케팅 검토 후 리뉴얼 결정, 1개월 만에 리뉴얼 개발 완수하여 26년 5월 런칭
  • 팀 리딩: 비즈니스 모델 구상, 개발 일정 조율, 인력 관리 등 프로젝트 운영 전반 관리. 초기 구축 4인 팀에서 시작해 법인 설립 후 6인 팀(대표이사 제외)으로 확대
